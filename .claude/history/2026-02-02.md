**API Endpoints:**

- `GET /bbjd/v1/billing/subscription` - Current user's subscription status
- `POST /bbjd/v1/billing/create-checkout` - Create Stripe checkout session
- `POST /bbjd/v1/billing/create-paypal-order` - Create PayPal order (lifetime)
- `POST /bbjd/v1/billing/capture-paypal-order` - Capture PayPal order
- `POST /bbjd/v1/billing/create-paypal-subscription` - Create PayPal subscription
- `GET /bbjd/v1/billing/portal` - Get Stripe customer portal URL
- `POST /bbjd/v1/billing/cancel` - Cancel subscription
- Webhook endpoints for Stripe and PayPal

### Key Implementation Details

1. **Supporter Detection:** Uses `user.user_roles` from AuthContext (not API call). Roles checked: `administrator`, `editor`, `supporter`, `lifetime`

2. **Stripe Flow:** Click button → Create checkout session → Redirect to Stripe hosted page → Webhook assigns role

3. **PayPal Flow:** Load SDK dynamically → Render PayPal buttons → Create order/subscription → Capture on approve → Role assigned

4. **API Credentials:** Stored in WordPress options via ApiSettingsPage (not wp-config.php constants)

5. **Billing FAQ:** 5 questions added to Settings Help tab
