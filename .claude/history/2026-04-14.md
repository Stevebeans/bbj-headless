## What was accomplished

### Root cause: `/compare/[matchup]` spider trap
- Vercel Functions observability showed `/compare/[matchup]` was 70%+ of invocations (4K invocations, 2.07 GB-Hrs in 24h)
- With ~500+ players, combinations are ~125k permutations — bots crawling every one
- `SuggestedComparisons` + `RelatedPlayers` feed crawlers 6+ new matchup URLs per page hit (emergent trap)
- No `/compare/` entry in robots.js disallow

### Fixes deployed to staging (commit 00048b8 on `staging` branch)
1. Added `/compare/` to `src/app/robots.js` disallow
2. Added `robots: { index: false, follow: false }` to all metadata return paths in `src/app/compare/[matchup]/page.jsx`

### Vercel Fluid Compute enabled
- User toggled Fluid Compute on in dashboard (was on legacy Function Duration pricing)
- Immediate impact visible: `/compare/` hits went from ~1.9s/invocation (legacy GB-Hrs) to ~123ms Active CPU — roughly 10-20x cost reduction per hit
- Legacy $14.07 Function Duration charge stays on this month's bill (sunk), but new usage flows to Fluid Active CPU / Provisioned Memory meters (both showing ~$0 so far)

### Ad strategy decisions
- User wants Al Jazeera-style cleaner UX: top leaderboard + sticky footer only for "hero" placements
- Identified that `bigbrotherjunkies_leaderboard_atf` already exists in Freestar — no new slot needed
- Confirmed the annoying "pop-up between pages" IS the `bigbrotherjunkies_google_interstitial` (Google's full-screen modal format)
- Revenue tradeoff from 2024: killing interstitial = ~$1,700/yr loss (~11%), acceptable
- Pushdown (`sticky_pushdown` ~$3,700/yr) will stay — user confirmed it's not annoying after seeing the illustration
