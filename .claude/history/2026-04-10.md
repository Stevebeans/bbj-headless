## Major Discovery: Vercel Cost Root Cause

### The Problem
Vercel Pro plan at 75% of $20 included credit ($15.08 consumed) with 11 days left in cycle. Function Duration was $12.54 — 83% of infrastructure cost. Average function invocation was 2.8 seconds (should be <0.5s for a static-first site).

### Root Cause (TWO layers)
1. **Root layout (`src/app/layout.jsx`) called `getInitialAuthState()` which invoked `cookies()`** — this opts the ENTIRE route tree into dynamic rendering in Next.js 15 App Router. Every single page was server-rendered on every request.
2. **10 pages had explicit `export const dynamic = "force-dynamic"`** — redundant given #1, but would still force dynamic after fixing #1.

### Secondary issues
- `src/lib/api/seasons.js` — getSeasons/getSeasonBySlug/getSeasonById missing explicit revalidate, defaulting to 60s
- `src/lib/utils/autoLink.js` — O(n²) isInsideTag() regex scanning, not cached with unstable_cache
- Season page's generateMetadata() calls getSeasonBySlug() redundantly (already called in page component)
- Zero `unstable_cache` usage anywhere in codebase
- CLAUDE.md self-contradicts: "100% static webhook-driven" vs "all fetches must have revalidate: 60"

## What Was Done

### Phase 3A — Pull auth out of root layout (COMMITTED + PUSHED)
**Commit:** `d67901c` on `staging` branch

**Files changed:**
1. `src/app/layout.jsx` — Removed `getInitialAuthState()` call and import. Layout is now fully static (no cookies/headers/draftMode calls). Bumped ad-settings revalidate from 60s to 3600s. Passes `initialShouldShowAds` (anonymous baseline) and `supporterRoles` to Providers instead of server-computed `shouldShowAds`.
2. `src/components/Providers.jsx` — Dropped `initialUser` prop, added `supporterRoles` prop, passes it through to AdProvider.
3. `src/context/AuthContext.jsx` — Removed `initialUser` parameter entirely. Now hydrates auth state purely client-side: reads `bbj_token` cookie (JWT decode for id/email/roles) + `bbj_user` cookie (cached avatar/name). Header shows a skeleton during the ~20-80ms hydration window (existing loading state in Header.jsx handles this).
4. `src/context/AdContext.jsx` — Accepts `supporterRoles` prop, computes `shouldShowAds = initialShouldShowAds && !isSupporter` client-side using `useAuth()`. All downstream effects (ad-blocker detection, route tracking, email passthrough) now key off computed `shouldShowAds` instead of the layout's `initialShouldShowAds`.

**Build result:** `next build` succeeded. Route table shows homepage and 30+ routes changed from `ƒ` Dynamic to `○` Static. Pages with explicit `force-dynamic` unchanged (addressed in Phase 3B).
