### Homepage Performance Fix (Critical)
- **Problem:** `wp/v2/posts?_embed` took 14s on staging (N+1 queries). 6 parallel API calls competing for PHP workers caused 18s wall time → Vercel 504 timeout.
- **Fix 1:** Created custom `/bbjd/v1/posts` endpoint — single WP_Query loop with direct lookups. 14s → 3s.
- **Fix 2:** Combined all 6 homepage calls into single `/bbjd/v1/homepage` endpoint. One PHP boot, one response, 60s transient cache (Redis-backed). 18s → 2.2s cold, 1.2s warm.
- Updated `page.jsx` to use `getHomepageData()` single call
- Individual endpoints still exist for non-homepage pages
