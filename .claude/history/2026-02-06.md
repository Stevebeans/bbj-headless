# 2026-02-06 — Admin Analytics Dashboard

## What Was Accomplished

Implemented the full Admin Analytics Dashboard (Stats tab) that pulls GA4 data via the Google Analytics Data API.

### New Files Created
- `wp-plugin: src/Api/AnalyticsRoutes.php` — 5 GA4 Data API endpoints with 15-min transient caching, `cachedReport` helper pattern
- `src/lib/api/analytics.js` — Frontend API client for all 5 analytics endpoints
- `src/app/admin/stats/page.jsx` — Full stats dashboard with 8 widget sections using Recharts

### Modified Files
- `wp-plugin: src/Admin/Pages/ApiSettingsPage.php` — Added GA4 Property ID + Service Account JSON fields
- `wp-plugin: src/Api/AdminRoutes.php` — Added `analytics_dashboard` to DEFAULT_PERMISSIONS
- `wp-plugin: src/Plugin.php` — Registered AnalyticsRoutes
- `src/app/admin/layout.jsx` — Added Stats tab with ChartIcon, permission-gated
- `src/hooks/useAdBlockDetector.js` — Fires `ad_blocker_detected` GA4 event

### Dependencies Added
- `recharts` (npm) — Chart components for Next.js
- `google/analytics-data` (composer) — GA4 Data API PHP client

### Dashboard Widgets (8 sections)
1. KPI Cards (Users, Page Views, Avg Session, Bounce Rate)
2. Traffic Over Time (line chart)
3. Top Pages + Top Referrers (tables)
4. Traffic Sources (horizontal bar) + Devices (donut)
5. Peak Hours (bar chart) + Top Countries (table)
6. Ad Blocker Detection (stat card + trend line)
7. Landing Pages (full-width table)
8. Content Type Breakdown (horizontal bar)

### Date Controls
- Preset buttons: 1D, 7D, 1M, 3M, 6M, 1Y
- Season dropdown (±14 days from season start/end)
- Custom date range inputs

## Build Status
- `npm run build` passes clean
- Stats page: 130 kB first load (expected for Recharts)

## Setup Required (One-Time)
1. Create Google Cloud service account with GA4 Data API access
2. Add service account email as viewer on GA4 property
3. Get numeric Property ID from GA4 Admin (format: `properties/XXXXXXXXX`)
4. Paste credentials in WP Admin → BBJ Data → API Settings → Google Analytics section

## Architecture
- Frontend fetches 5 API endpoints in parallel via `Promise.all`
- Backend uses `cachedReport` helper that handles caching/auth/errors for all endpoints
- REST transport (no gRPC needed — works on Cloudways)
- Permission: `analytics_dashboard` (administrator-only by default)
