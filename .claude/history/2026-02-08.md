### Supporter Status Not Updating After Subscription
**Problem:** After subscribing, supporter status didn't show across the site (header, quick reply, etc.) until logout/login/refresh. The `refreshUser()` function in AuthContext was silently failing because `GET /bbjd/v1/auth/me` endpoint NEVER EXISTED.

**Fix:** Created the missing `GET /bbjd/v1/auth/me` endpoint in `AuthRoutes.php`:
- Validates Bearer JWT token
- Returns fresh user data from DB (roles, avatar, rank, is_supporter)
- Handles expired/invalid tokens with proper error responses

**Root Cause Chain:**
1. `refreshUser()` → `fetchCurrentUser()` → `GET /bbjd/v1/auth/me` (404)
2. Error caught silently, logged to console
3. Frontend state never updated with new roles
4. Only page refresh (re-reads JWT/cookie cache) showed correct state
